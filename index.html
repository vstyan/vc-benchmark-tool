App Documentation

Stress Test Mechanics

The test saturates a CPU thread across three domains to simulate heavy multitasking:

  • FPU Stress: 1.5M iterations of algebraic operations testing the Floating Point Unit.
  • Recursive Logic: Recursive Fibonacci ($F_{30}$) to stress the call stack and branch predictor.
  • Memory/Cache: Bitwise operations on a 1M element buffer to stress L1/L2 cache and RAM bandwidth.

Jank Definition (33.3ms vs 16.7ms)

Standard UI jank is typically defined as 16.7ms (60Hz). However, this tool uses a relaxed threshold of 33.3ms.

This is intentional: we are measuring the jank while the system is running under heavy workloads. The goal is to evaluate if the system can maintain a stable 30 FPS (the baseline for smooth video conferencing) even during peak stress.

Note: A pulsating beep triggers if the system hits the Critical state to warn of thermal throttling.

System Efficiency

Measured as the percentage of time the system maintains a Nominal pressure state. This highlights thermal sustainability.
System Pressure

This metric uses the Compute Pressure API to monitor high-level hardware stress.

It factors in thermal constraints and power management to report how the overall system is coping.
Efficiency Logic

This score represents the Percentage of time in Nominal state.

A higher percentage indicates handled workload within energy-efficient thermal envelopes.
